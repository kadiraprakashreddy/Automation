2. Layout / Design Recommendations

Container

Center the form horizontally in the page/tab.

Max-width: 600–700px for readability.

Padding: 24px all around.

Background: subtle card style (box-shadow: 0 2px 8px rgba(0,0,0,0.1)).

Fields

Rule Name / Author / Description:

Label left-aligned, input box full width of container.

Rounded corners (4–6px), consistent height (~40px).

Placeholder text for guidance.

Version:

Displayed as read-only text or disabled input box.

Spacing

16–20px vertical spacing between fields.

Group all fields visually under “Rule Information” header.

Next Button

Centered horizontally below all fields.

Height: 40–44px, rounded corners (8px).

Primary color (e.g., blue).

Disabled until all required fields are filled (Rule Name, Author).

Hover effect for better UX feedback.

3. Optional Enhancements

Add tooltip/help icon next to fields for additional info.

Responsive: inputs stack vertically on small screens.

Inline validation: mark empty required fields with red border.

4. Example UI Flow

User enters Rule Name → input validated.

User enters Author → input validated.

Optional: enter Description.

Version displayed automatically (1.0.0).

Once required fields filled, Next button becomes enabled.

Click Next → navigates to Workflow tab.
